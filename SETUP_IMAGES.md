# Настройка пути к изображениям

## Проблема
Изображения не загружаются, потому что путь к папке с изображениями не настроен.

## Решение

### Вариант 1: Создать файл .env.local

Создайте файл `.env.local` в корне проекта `C:\hunslor\.env.local` со следующим содержимым:

```env
IMAGES_PATH=C:\Users\Вітання!\Desktop\pictr
DATABASE_URL="file:./prisma/dev.db"
JWT_SECRET=your-secret-key-change-in-production
```

**Важно:** Замените путь `C:\Users\Вітання!\Desktop\pictr` на реальный путь к вашей папке с изображениями.

### Вариант 2: Использовать автоматическое определение

Если папка `pictr` находится на рабочем столе, система попытается найти её автоматически по пути:
`C:\Users\ВАШЕ_ИМЯ_ПОЛЬЗОВАТЕЛЯ\Desktop\pictr`

### Проверка

1. Убедитесь, что папка с изображениями существует
2. Структура должна быть:
   ```
   pictr/
   ├── iPhone-17-Pro-Max/
   │   ├── 01-main.webp
   │   ├── Orange-256GB/
   │   │   └── 01-main.webp
   │   └── Silver-512GB/
   │       └── 01-main.webp
   └── MacBook-Pro-16/
       └── 01-main.webp
   ```

3. Перезапустите dev server после создания `.env.local`

### Отладка

Откройте консоль браузера (F12) и проверьте:
- Запросы к `/api/images/...` 
- Ошибки в консоли сервера

В логах сервера вы увидите:
- `IMAGES_BASE_PATH: ...` - путь к папке
- `Requested path: ...` - запрашиваемый путь
- `File exists: true/false` - существует ли файл
